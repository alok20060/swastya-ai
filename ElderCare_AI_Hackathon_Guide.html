Onboarding Flow — Screen by Screen

1
Splash Screen (2.5 sec)

App logo + name animates in. Background: gradient from primary blue to purple. No skip button.

2
Terms & Conditions

Simple, large font. Two buttons: "I Agree" (filled) and "Decline" (outlined). Must scroll to bottom to enable Agree button. Mention data privacy, 112 emergency auto-call consent.

3

Cards for: Camera (for report scanning), Location (GPS safety), Bluetooth (watch), Notifications. Each has icon, title, "why we need this" text, and a toggle. All required.

4
Google Sign-In

Large Google Sign-In button. Sub-text: "We use Google to securely save your health data."

5

Indian phone format (+91). 6-digit OTP via Firebase. Resend OTP after 30 sec countdown.

6

Two large cards: "I am an Elder User 👴" and "I am a Caretaker / Family Member 👨‍👩‍👧". Different onboarding path for each.

Screen 7a: Name, DOB, Gender, Blood Group, Height, Weight, Preferred Language.

Screen 7c: Family medical history (inherited diseases — same list + Cancer, Alzheimer's, Stroke, Parkinson's).
Screen 7d: Current medications (add multiple), known allergies, name of primary doctor, clinic/hospital name, preferred pharmacy.

8
Emergency Contacts (up to 5)

Name, Phone, Relationship (Son/Daughter/Spouse/Friend/Caretaker), set priority order. First contact = primary caretaker. Import from contacts allowed.

For each meal (Breakfast, Lunch, Dinner, Evening Snack): pick 5–6 commonly eaten dishes from a regional Indian food list. These become quick-select options for food logging. Region selector: South Indian / North Indian / Bengali / Gujarati etc.

Ayushman Bharat card number, private insurance policy number. "Skip for now" allowed.

Scan for BLE devices. Show list. User taps their watch. Pair and test heart rate reading. "Skip — use phone only" option.

13

Collect: resting heart rate, baseline SpO2, normal steps per day. User sits quietly for 60 sec while app records. This sets their personalized thresholds (not generic).

Health Alert System — Watch Vitals

Data Collected from BLE Watch

💓 Heart Rate

Beats per minute (BPM). Available on ALL BLE watches. Primary vital.

🩸 SpO2 / Blood Oxygen

Available on most watches ₹3000+. Critical for respiratory issues.

👣 Steps Count

Available everywhere. Used for activity + calorie burn.

😴 Sleep Data

Light/deep/REM cycles. Available on Noise, boAt, Mi band etc.

🌡️ Skin Temperature

Available on Samsung, Fitbit Sense, some Noise. Trend analysis.

😤 Stress Level

HRV-based stress score. Available on mid-range watches.

Three Alert Levels

🟢 Level 1
STABLE
  • HR: 60–100 BPM
  • SpO2: 95–100%
  • Stress: Low
  • Normal activity detected

Action: Nothing. All good.

🟡 Level 2
CAUTION
  • HR: 101–130 or 50–59
  • SpO2: 91–94%
  • High stress prolonged
  • No movement for 4 hrs

Action: Notification to user + all caretakers. App popup "Are you okay?" with 2-min response window.

🔴 Level 3
CRITICAL
  • HR: >130 or <50
  • SpO2: <90%
  • Possible fall detected
  • No response to Caution alert

Action: Auto-call 112 → Notify ALL contacts → Share live GPS location → Log event in medical history.

AI Prediction Model

What It Predicts

🩺 Diabetes Risk

Based on BMI, family history, age, blood glucose from reports.

💔 Cardiac Risk

Based on age, HR trends, cholesterol from reports, family history.

🧠 Alzheimer's Risk

Family history + age + sleep patterns.

🫁 Respiratory Risk

Based on SpO2 trends, smoking history, air quality (optional).

💊 Medication Reminder Compliance

Predict if user likely skipped medicine based on patterns.

🌡️ Fever / Infection Risk

Body temp trend + recent food log + activity drop.

Input Features for the ML Model

CategoryFeatures
DemographicsAge, gender, BMI, blood group, location (AQI risk)
Medical HistoryExisting conditions (encoded), medications, allergy count
Family HistoryParent/sibling diseases (binary flags per disease)
Watch VitalsAvg HR, min SpO2, sleep hours, avg steps, stress score (7-day rolling avg)
Food LogAvg daily calories, meal consistency score, hydration score
Medical ReportsBlood glucose, HbA1c, cholesterol, hemoglobin (extracted via OCR)

Model Choice: XGBoost (Recommended)

Why: Handles tabular/mixed data well, very fast to train, high accuracy, easy in Python, interpretable.

Medical Report OCR Pipeline

1
User scans report using phone camera (real-time) or uploads photo

Flutter camera plugin → capture image → send to backend as base64.

2
Backend runs OCR

Use pytesseract or Google Vision API (free tier) on Colab to extract raw text.

3
Parse key values using regex + Gemini

Extract: HbA1c, Blood Glucose, Cholesterol, Hemoglobin, BP values etc. Use Gemini to interpret unstructured text.

4
Store structured values in Firestore

Update user's medical profile. Re-run prediction model with new data. Show changes in health trends.

5
Display to user + caretaker

Show extracted values with normal range indicators. Flag anything out of range in red.

Unique Feature: "Ask AI About My Report"

After scanning, user can ask the chatbot: "My HbA1c is 7.2, is that bad?" — Gemini responds with context-aware, layman-friendly explanation in their language.

Medicine Ordering + Doctor Appointments

Auto Medicine Ordering

For hackathon demo, integrate with Tata 1mg API or Netmeds (or mock it). The system:

  • Tracks current medicine stock (user inputs quantity when adding medicine)
  • Estimates finish date based on dosage schedule
  • Sends notification 5 days before stock runs out: "Your Metformin will run out in 5 days. Order now?"
  • One-tap reorder to preferred pharmacy
  • Caretaker also gets notification and can approve/order on behalf of user

Doctor / Hospital Appointment Booking

For demo: mock appointment booking with user's saved doctor. Future: integrate with Practo API or Apollo 247. Features:

  • Saved doctor details from registration
  • Pick date/time from calendar widget
  • Auto-reminder set for 1 day + 1 hour before
  • Option to share recent health report with doctor before appointment
  • Post-appointment: prompt to upload new prescription/report

Food Log & Calorie Tracking

Flow After Meal Reminder

1
Meal reminder fires (e.g., Lunch at 1 PM)

Watch vibrates + phone notification.

2
1 hour later: "Did you have lunch?"

Yes / No / Had it early buttons. If No: notify caretaker. If Yes: go to step 3.

3
Quick food picker screen

Shows 5-6 pre-configured dishes from their region (set during registration). E.g., South Indian: Dosa, Rice+Sambar, Idli, Chapati, Meals, Upma. Plus "Others" option.

4
Quantity selector (simple)

Small / Medium / Large serving. Not grams — too complex for elderly.

5
Calories auto-calculated

From Indian food calorie database (CSV, pre-loaded). Stored in Firestore with timestamp.

6
Daily calorie balance shown

Calories in (food) vs Calories out (steps × MET factor from watch). If deficit: suggest eating more. If surplus + sedentary: suggest a walk.

Why This Data is Valuable

  • In emergencies, doctors need to know what was last eaten (drug interactions, blood sugar context)
  • Feeds into prediction model (poor nutrition = higher risk)
  • Helps detect eating disorders common in elderly (forgetting to eat)
  • Caretakers can monitor from their app
Unique Feature: If user reports feeling unwell + food log shows they skipped 2+ meals in a row, the AI automatically flags this as a risk factor and increases monitoring frequency.

Mobile App UI Design System

Design Principles for Elderly Indian Users

🔤 Large Text

Minimum 16sp body, 22sp+ for important data. Let users increase font size.

👆 Big Touch Targets

All buttons min 56dp height. Generous padding. No tiny icons.

🎨 High Contrast

Dark text on light backgrounds. No light gray on white. Pass WCAG AA.

🖼️ Visual over Text

Icons + color for health status. Don't rely on text alone. Use familiar symbols.

🔕 Reduce Clutter

Max 3–4 items per screen. One primary action per screen. Clean white cards.

🔊 Voice Option

Text-to-speech for notifications (elderly often hard of hearing). Use Flutter TTS package.

Screen-by-Screen UI Guide

🏠 Home Dashboard (User)

Top section: Greeting "Good Morning, Ramaiah Ji 🙏" + date. Large circular Recovery Score (0-100, color coded). Below: 4 quick-stat cards in 2x2 grid — Heart Rate, SpO2, Steps Today, Sleep. Below that: Today's Reminders list (next 3 upcoming). Bottom: SOS button (red, full width, always visible). Bottom nav: Home, Health, Reminders, Chat, Profile.

📊 Health Details Screen

Scrollable. Sections:

  • Live Vitals: HR gauge (0–200 arc), SpO2 percentage circle, Stress level bar
  • Today's Trends: HR line chart (24hr), SpO2 area chart (24hr)
  • Weekly Insights: Steps bar chart (7 days), Sleep quality bars, Calorie in vs out chart
  • Advanced Metrics: HRV trend, Body temp trend, Resting HR trend, Recovery score history
  • Risk Assessment: Horizontal bars for each disease risk (Low/Medium/High with color)

⏰ Reminders Screen

Two tabs: "Today" and "All". Each reminder is a card with: Icon (💊🍽️💧), Title, Time, Status (Pending / Done / Snoozed), Toggle on/off. FAB (+) to add custom reminder. Caretaker additions shown with a 👨‍👩‍👧 badge.

💬 Chatbot Screen

Full-screen chat UI. Bubble-style messages. Quick action chips at top: "Check vitals", "Log food", "Order medicine", "I feel unwell". Large input field. Mic button for voice input (speech-to-text). Gemini responses stream in real-time.

👴 Profile Screen

Avatar (Google profile photo). Personal details. Medical history summary. Emergency contacts list (editable). Medical reports gallery (tap to view, camera icon to add new). Language preference. Settings. Logout.

📍 Caretaker Dashboard (different role)

Shows linked elder's: Profile summary, Current location on map, Today's compliance (medicines taken Y/N, food log, water intake), Alert history (last 7 days), Quick contact button (call/WhatsApp). Multiple elders can be added in one caretaker account.

from fastapi import FastAPI, UploadFile, File
from fastapi.middleware.cors import CORSMiddleware
from pydantic import BaseModel
import uvicorn
from pyngrok import ngrok
import nest_asyncio
import google.generativeai as genai

app = FastAPI(title="SevaCare AI API")

app.add_middleware(CORSMiddleware, allow_origins=["*"],
  allow_methods=["*"], allow_headers=["*"])

# ─── MODELS ─────────────────────────────────────────
class VitalsData(BaseModel):
    user_id: str
    heart_rate: int
    spo2: float
    steps: int
    stress_score: float = 0.0
    skin_temp: float = 36.6

class ChatMessage(BaseModel):
    user_id: str
    message: str
    user_context: dict  # name, age, conditions, recent vitals etc

# ─── HEALTH ALERT ENDPOINT ──────────────────────────
@app.post("/health-check")
async def health_check(vitals: VitalsData):
    level = "STABLE"
    reasons = []

    if vitals.heart_rate > 130 or vitals.heart_rate < 50:
        level = "CRITICAL"
        reasons.append(f"Heart rate {vitals.heart_rate} BPM is dangerous")
    elif vitals.heart_rate > 100 or vitals.heart_rate < 60:
        level = max(level, "CAUTION") if level != "CRITICAL" else level
        reasons.append(f"Heart rate {vitals.heart_rate} BPM needs attention")

    if vitals.spo2 < 90:
        level = "CRITICAL"
        reasons.append(f"SpO2 {vitals.spo2}% is critically low")
    elif vitals.spo2 < 95:
        if level != "CRITICAL": level = "CAUTION"
        reasons.append(f"SpO2 {vitals.spo2}% is below normal")

    return {
        "status": level,
        "reasons": reasons,
        "action": get_action(level),
        "timestamp": datetime.now().isoformat()
    }

def get_action(level):
    if level == "STABLE": return "No action needed"
    if level == "CAUTION": return "NOTIFY_USER_AND_CARETAKERS"
    return "CALL_112_AND_NOTIFY_ALL"

import 'package:firebase_auth/firebase_auth.dart';

class AuthService {
  final FirebaseAuth _auth = FirebaseAuth.instance;
  final GoogleSignIn _google = GoogleSignIn();

  Future signInWithGoogle() async {
    final GoogleSignInAccount? googleUser = await _google.signIn();
    final GoogleSignInAuthentication googleAuth = 
        await googleUser!.authentication;
    final credential = GoogleAuthProvider.credential(
        accessToken: googleAuth.accessToken, idToken: googleAuth.idToken);
    return _auth.signInWithCredential(credential);
  }

  Future sendOTP(String phoneNumber) async {
    await _auth.verifyPhoneNumber(
      phoneNumber: '+91$phoneNumber',
      verificationCompleted: (cred) => _auth.signInWithCredential(cred),
      verificationFailed: (e) => print('OTP Error: $e'),
      codeSent: (String verificationId, int? token) {
        // Store verificationId, show OTP input screen
      },
      codeAutoRetrievalTimeout: (String verificationId) {},
    );
  }
}

BLE Service — Connect Watch & Read Heart Rate


// ble_service.dart
import 'package:flutter_blue_plus/flutter_blue_plus.dart';

class BLEService {
  static const HEART_RATE_UUID = "0000180d-0000-1000-8000-00805f9b34fb";
  static const HR_MEASUREMENT_UUID = "00002a37-0000-1000-8000-00805f9b34fb";

  Stream getHeartRateStream(BluetoothDevice device) async* {
    List services = await device.discoverServices();
    for (BluetoothService service in services) {
      if (service.uuid.toString() == HEART_RATE_UUID) {
        for (BluetoothCharacteristic char in service.characteristics) {
          if (char.uuid.toString() == HR_MEASUREMENT_UUID) {
            await char.setNotifyValue(true);
            yield* char.onValueReceived.map((val) => val[1]); // BPM value
          }
        }
      }
    }
  }
}

Geofence — Home Boundary Alert


// location_service.dart
import 'package:geolocator/geolocator.dart';

class LocationService {
  // Home coordinates saved during registration
  static double homeLat = 12.9716;
  static double homeLng = 77.5946;
  static double GEOFENCE_RADIUS = 25.0; // meters

  Stream monitorGeofence() {
    return Geolocator.getPositionStream(
      locationSettings: const LocationSettings(
        accuracy: LocationAccuracy.high,
        distanceFilter: 5,  // update every 5 meters of movement
      ),
    ).map((position) {
      double distance = Geolocator.distanceBetween(
        homeLat, homeLng,
        position.latitude, position.longitude,
      );
      return distance > GEOFENCE_RADIUS; // true = outside home zone
    });
  }
}

FastAPI — Prediction Endpoint


# In sevacare_api.ipynb

import joblib
import numpy as np

diabetes_model = joblib.load('diabetes_model.pkl')

class PredictionInput(BaseModel):
    glucose: float
    blood_pressure: float
    bmi: float
    age: int
    diabetes_pedigree: float

@app.post("/predict/diabetes")
async def predict_diabetes(data: PredictionInput):
    features = np.array([[0, data.glucose, data.blood_pressure,
                          20, 80, data.bmi, data.diabetes_pedigree, data.age]])
    probability = diabetes_model.predict_proba(features)[0][1]
    
    risk = "Low" if probability < 0.3 else "Medium" if probability < 0.6 else "High"
    return {
        "diabetes_risk": risk,
        "probability": float(probability),
        "recommendation": get_recommendation(risk)
    }

def get_recommendation(risk):
    if risk == "Low": return "Keep up your healthy lifestyle!"
    if risk == "Medium": return "Monitor blood sugar. Reduce sugar intake."
    return "Consult your doctor for diabetes screening soon."
